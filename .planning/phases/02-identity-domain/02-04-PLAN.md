<constraints>
- NO SSN (D-A). The backfill never creates or moves an SSN field. If a source entity
  somehow contains an ssn-shaped field, it is left in place and flagged, never copied
  into `identity`.
- Dry-run FIRST: the default invocation makes no writes. Persisting requires an
  explicit `--apply` (or `--no-dry-run`) flag, matching the `migrate_financial_v2.py`
  pattern (default dry-run safe).
- Idempotent: re-running after a successful move detects the entity already in
  `identity` and makes no further change.
- Reversible: record the source domain/path and destination so a move can be undone;
  do not hard-delete source data without preserving a record (prefer copy-then-mark
  or emit a reversible mutation event).
- Reuse the existing PKM service read/write API â€” do not write raw SQL that bypasses
  the BYOK-encrypted domain payload contract.
- vault-pkm-governance applies (PKM data-boundary change): run governance gates at
  the human checkpoint.
</constraints>
