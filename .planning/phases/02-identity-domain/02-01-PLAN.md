<objective>
Register `identity` as a canonical PKM domain and make the agent's add/update
classification route PII (name, email, address, dob, phone) to `identity` while
forcing every identity write to `confirm_first`. This is the critical path that
resolves the phase 01 UAT failure ("update my address" auto-saved to `financial`
with no confirmation).

Purpose: Build the missing *producer* for the identity domain. The KYC consumer
already reads `attr.identity.*` (`one_email_kyc_service.py:56`) â€” nothing produces
it today. The registry is the single source of truth; adding `identity` there
auto-surfaces it to the LLM ontology and the fully-dynamic frontend.

Output: `identity` in the canonical registry + both classifiers + planner prompts,
the disavowal removed from agent_chat_service, the normalizer force-confirm, and
regression tests including the exact UAT case.
</objective>
