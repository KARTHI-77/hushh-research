<objective>
Build the backend redact-safe LLM proxy endpoint `POST /api/one/kyc/workflows/{id}/redraft-llm`.
It accepts a PII-free tokenized template + instruction from the client, validates the
`agent.kyc.redraft.llm` consent scope, forwards the payload to server-side Gemini Vertex
(reusing the existing `_gemini_client` from `operons/kai/llm.py`), and returns the rewritten
template. It never persists or logs the request body; `draft_body` stays NULL.

Purpose: The client (Wave 3) needs a server endpoint it can call without exposing real PII
to the network. This endpoint is the only path between the browser and Gemini for KYC
redrafts. Its no-persist / no-log contract is the enforcement layer for D-D. The endpoint
mirrors the `analyze_stock_with_gemini` pattern in `operons/kai/llm.py` exactly: same Vertex
client, same `validate_token` consent gate (different scope), same model/temperature
constants.

Output: One new Pydantic schema `LlmRedraftRequest`, one new route handler
`one_kyc_redraft_llm`, and one new async service method `redraft_llm` in
`one_email_kyc_service.py`. No new Vertex client, no new pip packages, no DB column.
</objective>
