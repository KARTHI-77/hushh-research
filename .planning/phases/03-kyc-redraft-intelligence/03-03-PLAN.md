<objective>
Build the client-side redact → rewrite → re-fill pipeline. Helper functions go into
`one-kyc-client-zk-service.ts` alongside `buildDraft`: `redactDraftForLlm` (deterministic
PII redaction from approvedValues), `resubstituteDraft` (token re-fill),
`validateTokenIntegrity` (hard integrity check), and `htmlFromPlaintext` (re-derive preview
HTML from the LLM-rewritten plaintext). A new `isKeywordOnlyInstruction` detector drives
routing. `OneKycService.redraftWithLlm` is the thin HTTP client for Wave 2's endpoint. The
`runAction("redraft")` handler in `page.tsx` is extended with an LLM branch that calls the
full pipeline; the existing regex path is kept as the fast local path for keyword instructions.

Purpose: No real PII value ever leaves the browser (D-B). Redaction is deterministic because
`approvedValues` already maps field keys to their string values — iteration is exact. The LLM
rewrites the PII-free plaintext only (plaintext-then-re-render is safest for token integrity
per RESEARCH §6); the preview html is re-derived from the rewritten plaintext so the user
actually sees the LLM result (DraftReplyPreview at page.tsx:2298 renders htmlBody via
dangerouslySetInnerHTML — leaving the stale render-model html would show the OLD draft). Token
integrity is a hard gate: a mangled LLM response never silently replaces the draft. The
injected `KycLlmRewriteCallable` interface keeps the pipeline target-agnostic (D-G).

Output: Five new functions + one type in `one-kyc-client-zk-service.ts`; one new static method
in `one-kyc-service.ts`; extended `runAction` handler in `page.tsx`. No new npm packages.
</objective>

- D-B (trust model): `redactDraftForLlm` MUST iterate the `approvedValues` record from
  `KycDraftBuildResult` — not regex-strip the rendered body — so redaction is exact and
  complete. Every value in `approvedValues` must appear in `tokenMap`.
- D-H guardrail 1 (redaction completeness): assert that after redaction, no value from
  `approvedValues` appears verbatim in `tokenizedTemplate`. If any value with length >= 3
  is found verbatim in the tokenized output, throw before sending.
- D-H guardrail 2 (token integrity): `validateTokenIntegrity` must check each token in
  `tokenMap` appears in `rewrittenTemplate` EXACTLY once (not zero, not two or more) and
  that no token appears in `rewrittenTemplate` that is NOT in `tokenMap`. Return false on
  any violation — never throw silently past it.
- D-H guardrail 3 (field re-validation): after re-substitution, feed the original export
  payloads to `buildDraft` and compare its `approvedValues` keys to the pre-redraft keys.
  The returned `approvedValues` must have the same keys as before (no field added, dropped,
  or renamed). If keys differ, fall back to the prior draft.
- PREVIEW FIDELITY (Blocker fix): The LLM rewrites the plaintext body only. After
  re-substitution, the preview `htmlBody` MUST be re-derived from the resubstituted plaintext
  via `htmlFromPlaintext(resubstitutedBody)`. Do NOT reuse `revalidatedDraft.htmlBody` (it is
  rebuilt from the original render model with no LLM input, so the user would see the old
  draft). `htmlFromPlaintext` MUST HTML-escape its input (escape &, <, >, ", ') and convert
  newlines to paragraph/line-break markup so the LLM output is rendered safely.
- D-G (forward-compat): The LLM call is made through a `KycLlmRewriteCallable` type
  (a function type: `(tokenizedTemplate: string, instruction: string) => Promise<string>`)
  passed into the branch, not hardcoded. `OneKycService.redraftWithLlm` is the default
  implementation; an on-device implementation can replace it without touching the pipeline.
- D-F (backward compat): `redraftTransformFromInstructions` and the existing regex path
  MUST remain unchanged and reachable. No modification to the function itself.
- No new npm packages. No changes to `buildApprovedDisclosurePlainText`,
  `buildApprovedDisclosureHtml`, or `redraftTransformFromInstructions`.

    4. `export function resubstituteDraft(tokenizedTemplate: string, tokenMap: Record<string, string>): string`
       Replace each `{{Fi}}` placeholder with `tokenMap[Fi]`. Iterate keys of tokenMap in
       insertion order. Return the substituted string. If a placeholder in the template has
       no entry in tokenMap, leave it as-is (the integrity check catches this upstream).

    5. `export function validateTokenIntegrity(tokenizedTemplate: string, rewrittenTemplate: string, tokenMap: Record<string, string>): boolean`
       For each key `k` in tokenMap:

           `/\{\{k\}\}/g`.
         - If count !== 1, return false.
       Additionally, find ALL `{{...}}` tokens in `rewrittenTemplate` via `/\{\{([^}]+)\}\}/g`.
       If any matched token key is NOT in tokenMap, return false.
       Return true only if all checks pass.
