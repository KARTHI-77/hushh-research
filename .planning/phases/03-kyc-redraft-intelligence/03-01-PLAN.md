<objective>
Add the `agent.kyc.redraft.llm` optional consent scope to the backend scope registry and
document the narrow ZK carve-out in the KYC agent manifest. This is the gating prerequisite
for all subsequent waves: the proxy endpoint (Wave 2) validates this scope before forwarding
anything to Gemini, and the frontend (Wave 3) gates the LLM path behind it.

Purpose: The ZK carve-out (D-D) must be explicit and auditable. A manifest flag
`llm_redraft_tokenized: true` records that the backend transiently sees a PII-free template
— never a real value — and never persists or logs the body. Without this flag the carve-out
is implicit and unauditable. Without the scope, the proxy endpoint has no consent gate.

Output: One new `ConsentScope` enum member in `constants.py`; one new entry in
`manifest.py` `optional_scopes`; one new `llm_redraft_tokenized` capability block in
`manifest.py`. No other files changed.
</objective>
