## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM tool call → executePkmUpdateTool | `toolEvent.slots` fields arrive as `unknown` from SSE parser; all four D-07 slots must be type-guarded before use |
| `handleSavePkmReview` update branch → PkmWriteCoordinator | The write uses `context.currentDomainData` (authoritative) inside the `build` callback, not the LLM's `current_value` (display-only) |
| Vault key → write call | `vaultKey` and `token` must be checked before any call to `saveAgentPkmUpdate` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | `executePkmUpdateTool` slot reads | mitigate | All four slots read with `typeof x === "string" ? x.trim() : ""` guard — per slot-access-guard pattern in PATTERNS.md. Malformed or missing slots default to empty string, triggering the "no review cards" path rather than a partial write |
| T-03-02 | Tampering | `handleSavePkmReview` update branch | mitigate | Vault guard `(!vaultKey || !token)` fires before `saveAgentPkmUpdate` call; `PkmWriteCoordinator` uses `context.currentDomainData` (fresh optimistic-lock read) not the LLM-supplied `current_value` slot |
| T-03-03 | Elevation of Privilege | `pkm.update` early-exit guard ordering | mitigate | Guard is placed BEFORE `executeAgentGatewayAction` — same position as `pkm.add` guard. This ensures the gateway (which has `router.push` dispatch) never receives a `pkm.update` action ID (per D-02) |
| T-03-04 | Denial of Service | `setActivePkmToolCount` not decremented | mitigate | `executePkmUpdateTool` wraps all logic in `try/finally` with `setActivePkmToolCount((c) => Math.max(0, c - 1))` in `finally` — copied from `executePkmAddTool` pattern |
| T-03-SC | Tampering | npm/pip installs | accept | No new packages installed in this plan |
</threat_model>
