# Report Artifact Generation

Use this workflow pack when the task matches `report-artifact-generation`.

## Goal

Create a report-style artifact that summarizes repo-backed truth without becoming a second source of truth.

## Steps

1. Start with `docs-governance` and use `founder-brief-curation` for artifact voice and shareability rules.
2. Classify the output as a report artifact, not a canonical docs replacement.
3. Link to canonical docs instead of copying long contracts.
4. Run docs verification and document any residual proof gap.

## Common Drift Risks

1. duplicating canonical docs in report form
2. unclear current-state versus future-state wording
3. missing verification note
